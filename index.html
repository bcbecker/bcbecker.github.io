Who am I?

As the big letters say, my name is Bryce and I'm a backend developer. Coding has been a part of my life since I was 18, tinkering with Arduino microcontrollers to build data monitoring systems for race cars. This was followed by using Python as a cancer research student to find meaning in collected data. Now, I've gone full-tilt into developing backend systems in any language necessary. I love learning new techniques and best practices wherever I can, so please don't hesitate to provide feedback. Shoot me an email, check out my Github, or connect with me on LinkedIn!

What can I do?

As a backend developer, I strive to create apps rooted in security and scalability. I believe the technologies used to solve a given problem are merely tools with which a developer applies their knowledge. Thus, I value my skills in a language-agnostic way, preferring to understand why we need a load balancer with sticky sessions instead of how to configure nginx to do so. Nonetheless, I've listed tools that I most often reach for to solve problems.
